编译Apollo源代码编译docker部署

Aug 16 2019

1.源码下载

访问https://github.com/ctripcorp/apollo官方网站下载源码。

2.项目构建

解压源代码导入到IDEA，构建项目，需要依赖Maven环境。

3.配置修改

修改apollo-portal/src/main/resources/apollo-env.properties中的local.meta=http://192.168.6.32:18080
修改scripts/build.bat(sh)文件中的数据配置，以及dev_meta=http://192.168.6.32:18080，然后执行该文件

4.发布构建

apollo-adminservice、apollo-configservice、apollo-portal模块中的target的文件及Dockfile拷贝到如下目录
apollo
├── apollo-adminservice
│ ├── apollo-adminservice-1.5.0-SNAPSHOT-github.zip
│ └── Dockerfile
├── apollo-configservice
│ ├── apollo-configservice-1.5.0-SNAPSHOT-github.zip
│ └── Dockerfile
│ ├── apollo-configservice-1.5.0-SNAPSHOT-github.zip
│ └── Dockerfile
└── apollo-portal
├── apollo-portal-1.5.0-SNAPSHOT-github.zip
└── Dockerfile

5.doker部署

将步骤4的文件上传到有docker环境的服务器，然后分别执行如下命令
cd apollo-configservice
docker build -t apollo-configservice .
docker run -p 18080:8080 -d –name apollo-configservice apollo-configservice
或者部署到Portainer
docker tag apollo-configservice:latest hub.uupt-docker.com/local-test/apollo-configservice:latest
docker push hub.uupt-docker.com/local-test/apollo-configservice:latest

cd apollo-adminservice
docker build -t apollo-adminservice .
docker run -p 18090:8090 -d –name apollo-adminservice apollo-adminservice
或者部署到Portainer
docker tag apollo-adminservice:latest hub.uupt-docker.com/local-test/apollo-adminservice:latest
docker push hub.uupt-docker.com/local-test/apollo-adminservice:latest

cd apollo-portal
docker build -t apollo-portal .
docker run -p 18070:8070 -d –name apollo-portal apollo-portal
或者部署到Portainer
docker tag apollo-portal:latest hub.uupt-docker.com/local-test/apollo-portal:latest
docker push hub.uupt-docker.com/local-test/apollo-portal:latest

注意启动顺序，configservice -> adminservice -> portal，为了避免常规冲突，我们在默认的端口上＋10000。
